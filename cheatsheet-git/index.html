Git Cheat Sheet

Git é o sistema de controle de versão distribuído gratuito e de código aberto responsável por tudo o que o GitHub relacionado que acontece localmente no seu computador. Esta folha de dicas apresenta as informações mais importantes e comumente usei comandos do Git para fácil referência.


INSTALAÇÃO E GUIS

Com instaladores específicos da plataforma para o Git, o GitHub também fornece o facilidade de manter-se atualizado com as últimas versões do comando ferramenta de linha enquanto fornece uma interface gráfica do usuário para o dia-a-dia interação, revisão e sincronização do repositório.

STAGE & SNAPSHOT

Trabalhando com snapshot e a área de preparo do Git

-------------------------------

git status
mostre arquivos modificados no diretório de trabalho, preparados para seu próximo commit

git add [file]
adicione um arquivo como ele se parece agora ao seu próximo commit (estágio)

git reset [file]
remover o estágio de um arquivo enquanto retém as alterações no diretório de trabalho

git diff
diff do que foi alterado, mas não encenado

git diff --staged
diff do que está preparado, mas ainda não commitado

git commit -m “[descriptive message]”
confirme seu conteúdo preparado como um novo commit com descrição sobre a tarefa

SETUP

Configurando as informações do usuário usadas em todos os repositórios locais

-------------------------------

git config --global user.name “[firstname lastname]” defina um nome que seja identificável para crédito ao revisar o histórico da versão

git config --global user.email “[valid-email]”
defina um endereço de e-mail que será associado a cada marcador de histórico

git config --global color.ui auto
defina a coloração automática da linha de comando para o Git para facilitar a revisão

BRANCH & MERGE

Isolando o trabalho nas ramificações, alterando o contexto e integrando as alterações

-------------------------------

git branch
liste seus ramos. um * aparecerá ao lado do ramo atualmente ativo

git branch [branch-name]
criar uma nova ramificação no commit atual

git checkout
mude para outro ramo e verifique-o em seu diretório de trabalho

git merge [branch]
mesclar o histórico da ramificação especificada com o atual

git log
mostra todos os commits no histórico do branch atual

SETUP & INIT

Configurar informações do usuário, inicializar e clonar repositórios

-------------------------------

git init
inicializar um diretório existente como um repositório Git

git clone [url]
recuperar um repositório inteiro de um local hospedado via URL

INSPECT & COMPARE

Examinando logs, diffs e informações de objetos

-------------------------------

git log
mostra o histórico de commits para o branch atualmente ativo

git log branchB..branchA
mostre os commits no branchA que não estão no branchB

git log --follow [arquivo]
mostre os commits que alteraram o arquivo, mesmo entre renomeações

git diff branchB...branchA
mostre a diferença do que está na branchA que não está na branchB

git show [SHA]
mostre qualquer objeto no Git em formato legível por humanos

RASTREANDO MUDANÇAS DE PATH

Remoções de arquivo de controle de versão e alterações de caminho

-------------------------------

git rm [file]
exclua o arquivo do projeto e prepare a remoção para confirmação

git mv [existing-path] [new-path]
alterar um caminho de arquivo existente e preparar a mudança

git log --stat -M
mostre todos os logs de confirmação com indicação de todos os caminhos que foram movidos

SHARE & UPDATE

Recuperando atualizações de outro repositório e atualizando repositórios locais

-------------------------------

git remote add [alias] [url]
adicione um URL git como um alias

git fetch [alias]
buscar todas as ramificações desse controle remoto do Git

git merge [alias]/[branch]
mesclar uma ramificação remota em sua ramificação atual para atualizá-la

git push [alias] [branch]
Transmitir commits da ramificação local para a ramificação do repositório remoto

git pull
buscar e mesclar quaisquer commits da ramificação remota de rastreamento

REESCREVER O HISTÓRIICO

Reescrevendo branches, atualizando commits e limpando o histórico

-------------------------------

git rebase [branch]
aplicar quaisquer commits do branch atual antes do especificado

git reset --hard [commit]
limpar a área de preparação, reescrever a árvore de trabalho do commit especificado

COMMITS TEMPORÁRIOS

Armazene temporariamente arquivos rastreados e modificados para alterar as ramificações

-------------------------------

git stash
Salvar alterações modificadas e preparadas

git stash list
listar a ordem da pilha de alterações de arquivos armazenados

git stash pop
escrever trabalhando do topo da pilha de stash

git stash drop
descarte as alterações do topo da pilha de stash

IGNORAR PADRÕES

Evitando a preparação ou confirmação não intencional de arquivos

-------------------------------

logs/ *.notes pattern*/
Salve um arquivo com os padrões desejados como .gitignore com uma string direta correspondências ou globs curinga.

git config --global core.excludesfile [file]
padrão de ignorar em todo o sistema para todos os repositórios locais
